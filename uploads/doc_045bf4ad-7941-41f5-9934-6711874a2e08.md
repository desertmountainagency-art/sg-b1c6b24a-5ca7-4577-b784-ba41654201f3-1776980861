# **Future Self Mentor — Full-Stack Application Development Prompt**

## **Project Overview**

Build a full-stack web application designed for human growth and personal development. The core feature is an AI-powered mentor that embodies the user's future, idealized self — a version of them who has already achieved their goals, overcome their struggles, and integrated their lessons. The mentor speaks from lived experience, offering guidance, challenge, and support through a conversational interface.

---

## **System Architecture**

### **Frontend**

●     React or Next.js application with a clean, distraction-free interface  
●     Conversational UI that feels intimate and personal — not like a support chatbot  
●     User authentication and onboarding flow that collects the user's name, goals, current challenges, and emotional baseline  
●     Session history view showing past conversations and insights  
●     Goal tracker showing progress on goals the user has discussed with the mentor  
●     Journaling or reflection prompts generated by the AI based on past conversations  
●     Settings allowing users to adjust mentor tone preference: supportive, direct, balanced

### **Backend**

●     Node.js or Python API handling conversation logic and session management  
●     Integration with OpenAI API (or other LLM provider) using the mentor persona prompt as the system message  
●     User profile database storing name, goals, emotional baseline, conversation history, and progress markers  
●     Vector database (Pinecone, Weaviate, or similar) storing long-term memory of past conversations for context retrieval  
●     Session memory management ensuring the mentor remembers past discussions and references them naturally  
●     Goal tracking system that extracts commitments and action items from conversations

### **AI Mentor Integration**

●     The core behavior is governed by the mentor persona prompt, which must be injected as the system message for every conversation  
●     The mentor must always know the user's name and refer to them by name throughout the conversations  
●     The mentor must maintain continuity across sessions, referencing past conversations, goals, and emotional states  
●     The mentor must adapt to the user's emotional state in real-time, switching between supportive and direct modes as needed  
●     The mentor must avoid emojis, line dividers, and overly formal coaching language — it must sound like the user, fully realized

---

## **User Experience Flow**

### **Onboarding**

1. User signs up with email or social login  
2. User enters their name  
3. User completes a brief intake survey covering:

   ○     Current goals (short-term and long-term)

   ○     Current challenges or struggles

   ○     Emotional baseline (e.g., stressed, stuck, motivated, uncertain)

   ○     Preferred mentor style (warm and supportive, direct and challenging, or balanced)

4. System generates initial mentor introduction message, speaking from the future self perspective, acknowledging where the user is and where they're headed

### **Conversation Interface**

●     Clean, chat-style interface with a text input field  
●     Mentor initiates with a calibration question: "Want to talk this out like a friend, or do you want the full plan?"  
●     User responds, and mentor adapts length and tone accordingly  
●     Mentor uses past tense when giving advice: "Here's what I did" rather than "Here's what I would do"  
●     Mentor asks no more than three questions per response, allowing the user to answer before continuing  
●     Mentor tracks goals mentioned and periodically checks in on progress  
●     Mentor recognizes patterns across sessions and calls them out when necessary

### **Session History**

●     Users can view past conversations organized by date or topic  
●     Each session entry includes key insights or action items extracted by the AI  
●     Users can mark action items as complete, which updates the mentor's knowledge in future sessions

### **Goal Tracker**

●     System extracts goals and commitments from conversation text using NLP  
●     Goals are displayed in a dashboard with status: not started, in progress, completed  
●     Mentor references goal progress in future conversations, celebrating wins or addressing stalls

### **Reflection Prompts**

●     Based on conversation history, the system generates optional journaling prompts  
●     Prompts are offered at the end of each session or via daily notifications  
●     Mentor may reference past reflections in future conversations

---

## **Technical Requirements**

### **Authentication**

●     Secure user authentication using JWT or OAuth  
●     User profiles stored in PostgreSQL or MongoDB  
●     Sensitive data encrypted at rest

### **Conversation Management**

●     Each message sent and received is stored in the database linked to the user and session ID  
●     Long-term memory handled via vector embeddings: past conversations embedded and retrieved based on relevance to current message  
●     Mentor system prompt must be dynamically injected with user name and current context before each API call

### **AI Integration**

●     OpenAI API (GPT-4 or equivalent) used for conversation generation  
●     System prompt includes:  
○     Mentor persona instructions  
○     User name  
○     Current goals  
○     Recent conversation context retrieved from vector database  
●     Temperature and token limits tuned for natural, concise responses  
●     Rate limiting implemented to prevent abuse

### **Data Storage**

●     PostgreSQL for structured data: users, goals, sessions, action items  
●     Vector database for unstructured conversation memory  
●     Optional: Redis for session caching and real-time state

### **Notifications**

●     Optional push notifications or email reminders for:  
○     Daily check-in prompts  
○     Goal progress reminders  
○     Reflection prompts

## **Summary**

Build a full-stack application centered on an AI mentor that speaks as the user's future self. The mentor must embody wisdom, lived experience, and personal connection. The app must support onboarding, goal tracking, session history, and long-term memory. The AI must always speak in the past tense, adapt to emotional state, and use NLP and Socratic questioning to guide the user toward self-revelation and action. The interface must be clean and intimate, the backend must support memory and context retrieval, and the system must feel less like a chatbot and more like a conversation with the person the user is becoming.
